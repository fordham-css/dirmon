About

dirmon is a project designed to provide notifcations and rich insights based on the monitoring of command line behavior and access. dirmon is hosted by Fordham University's Computer Science Society.

Intentions

The original intention for dirmon is to provide actionable monitoring data to a very granular level. Like git, dirmon is intended to be easy to initialize and use, and virtually invisible so that it does not get in the way of a working directory. Using some machine learning elements, dirmon will eventually be able to detect suspicious or malicious behavior, and respond accordingly through notification as well as a "quarantine" filesystem backup procedure that will automatically begin backing up concerned directories.

The long term goal is to provide a tool for intelligent monitoring that anyone can use.

How it Works

dirmon scans the history files that are stored in every session of bash/terminal tty. Given the current directory it is tracking, it sorts through the command line history for any commands associated with the current directory it is tracking. It then notifies the user via popup of any detections.

dirmon logs bash activity to the hidden files /.dirmon and /.dirmon_origin in the directory you are monitoring. The logs are what the dirmon program uses to compare and detect for anomalies.

Installation and Usage

To use, just run the command dirmon inside the directory you wish to monitor!

To install, run the install.sh script. If you have errors installing, try the following commands:


  $ chmod +x install.sh
  $ ./install.sh

To uninstall, run the uninstall.sh script. If you have errors uninstalling, try the following commands:


  $ chmod +x uninstall.sh
  $ ./uninstall.sh

The uninstall command will clean dirmon from your computer!
